Detailed Description

template<typename Number>
class VectorMappings< Number >

Due the generalized nature of PRISMS-PF and how we handle the solving of PDEs on the backend, we end up with a slew of vectors that have to be managed and updated as we solve fields. While we can stored these in hierarchical data structure to preserve the logic and intuition this becomes unwieldy on the backend and is less performant than storing them in a flat structure or vector blocks.

This class has a unified description of these mappings so that everywhere else in the code we can simply use 1D vector indices.
